# Memory Manager - 记忆管理

## 功能

这个技能帮助 AI 助手更好地记住与用户的对话和重要的事情：

1. **每日笔记** - 记录当天发生的对话和事件到 `memory/YYYY-MM-DD.md`
2. **长期记忆** - 将重要信息整理到 `MEMORY.md` 供后续会话使用
3. **记忆搜索** - 回顾之前的对话内容和已保存的信息
4. **记忆整理** - 定期清理和更新 MEMORY.md，保持记忆的相关性

## 使用场景

- 用户说"记住这个"或"记下来"
- 需要回顾之前的对话内容
- 保存用户偏好、重要决策、项目进展
- 在心跳检查中整理记忆文件

## 文件结构

```
workspace/
├── MEMORY.md           # 长期记忆（跨会话）
├── memory/
│   ├── 2026-03-24.md   # 每日笔记
│   └── 2026-03-23.md
└── ...
```

## 工作流程

### 1. 记录当日事件

当对话中有值得记住的内容时，写入当天的记忆文件：

```markdown
<!-- memory/2026-03-24.md -->
# 2026-03-24

## 对话记录

### 13:30 - 技能创建讨论
- 用户希望我增加记忆功能
- 决定创建 memory-manager skill
- 目标：记住对话、保存偏好、回顾历史

### 15:00 - 项目进展
- 完成了 xxx 功能的开发
- 用户决定采用方案 B
```

### 2. 更新长期记忆

对于需要跨会话保留的重要信息，更新 MEMORY.md：

```markdown
<!-- MEMORY.md -->
# 长期记忆

## 用户偏好
- 喜欢简洁直接的回答
- 工作时段：12:00-22:00
- 讨厌长篇大论

## 重要项目
- 彩票店公众号开发中（审核中）
- OpenClaw 技能扩展计划

## 技术栈
- Windows 环境
- 使用 OpenClaw 管理 AI 模型
```

### 3. 回顾记忆

当用户问"我们昨天聊了什么"或"之前说过..."时：
- 先搜索 memory/ 目录下的相关文件
- 使用 memory_search 工具查找关键词
- 使用 memory_get 读取具体内容

## 记忆内容分类

**写入每日笔记 (memory/YYYY-MM-DD.md)：**
- 当天的对话摘要
- 临时性的讨论和想法
- 具体的工作进展

**写入长期记忆 (MEMORY.md)：**
- 用户偏好和习惯
- 重要的决策和结论
- 长期项目的状态
- 需要跨会话记住的信息

## 最佳实践

1. **及时记录** - 对话中遇到重要信息立即写入
2. **定期整理** - 每隔几天回顾每日笔记，提炼到 MEMORY.md
3. **保持简洁** - 只记录真正重要的内容，避免信息过载
4. **结构化存储** - 使用标题、列表等格式便于后续查找

## 脚本工具

### scripts/save_memory.js

快速保存记忆内容的辅助脚本：

```bash
node scripts/save_memory.js --type daily --content "今天讨论了 xxx"
node scripts/save_memory.js --type longterm --content "用户偏好：xxx"
```
