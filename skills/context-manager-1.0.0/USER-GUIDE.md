# Context Manager 用户指南

## 这是什么？

Context Manager 是一个智能上下文管理技能，当对话太长导致 AI 性能下降时，它会自动帮你保存重要信息，然后开启新的清爽对话。

## 为什么需要这个？

AI 有上下文限制（通常 32K-128K tokens），当对话太长时：
- AI 响应变慢
- 可能忘记前面的内容
- 性能下降

这个技能帮你解决这个问题，让你可以无限长对话！

## 如何使用？

### 三个简单命令

1. **检查状态** - 问："查一下上下文还剩多少"
   - AI 会告诉你当前使用率
   - 如果超过 85%，会建议清理

2. **开始整理** - 说："开始上下文整理"
   - AI 会自动提取关键信息
   - 生成总结文件（约 96% 压缩率）
   - 询问是否开启新会话

3. **继续对话** - 说："读取总结继续"
   - 在新会话中加载之前的总结
   - 无缝继续之前的对话

### 自动触发

当上下文使用率超过：
- **85%**：AI 会提醒你考虑清理
- **90%**：AI 会建议立即清理

## 工作原理

1. **监控**：实时跟踪上下文使用率
2. **总结**：提取关键信息（决策、待办、偏好等）
3. **保存**：生成总结文件到 `memory/context-summaries/`
4. **重置**：开启新会话，加载总结继续

## 示例流程

```
用户：查一下上下文还剩多少
AI：当前上下文使用率 92%，达到临界阈值，建议立即清理。

用户：开始上下文整理
AI：正在生成总结... 已完成！总结文件已保存。
     是否开启新会话继续对话？

用户：好的
AI：已开启新会话，之前的总结已加载。我们可以继续了！
```

## 文件位置

- 总结文件：`memory/context-summaries/context-summary-YYYY-MM-DD-HHMM.md`
- 技能文件：`skills/context-manager-1.0.0/`

## 常见问题

### Q: 总结会丢失信息吗？
A: 不会，只会提取关键信息（决策、待办、偏好、重要事实等），压缩率约 96%。

### Q: 可以手动触发吗？
A: 可以，任何时候说"开始上下文整理"即可。

### Q: 总结文件保存多久？
A: 默认保留最近 10 个总结，旧的会自动清理。

### Q: 会影响当前操作吗？
A: 不会，AI 会先询问确认，不会中断你的工作。

## 技术细节

- 阈值配置：85% 警告，90% 临界
- 总结格式：Markdown，包含所有关键信息
- 压缩率：通常 90-98%
- 兼容性：与所有 AI 模型兼容
