# Cross-Model Memory - 跨模型记忆一致性

## 问题背景

用户可能频繁切换模型（Kimi ↔ DeepSeek ↔ Qwen 等），但每个模型的对话历史是独立的。这导致：
- 昨天用 Kimi 聊的内容，今天用 DeepSeek 找不到
- 模型切换 = 记忆断层

## 解决方案

通过统一的记忆文件系统，让所有模型共享同一份记忆。

## 工作流程

### 1. 会话启动时加载记忆

每次会话开始，自动执行：

```
1. 读取 memory/ 目录下的今日笔记
2. 读取 MEMORY.md 长期记忆
3. 搜索最近 3 天的记忆文件
4. 如果有"昨天/之前/上次"等关键词，主动搜索相关记忆
```

### 2. 对话中实时保存

关键节点自动保存：

```
- 用户说"记住这个" → 立即保存到长期记忆
- 做出重要决策 → 保存到当日笔记
- 对话结束 → 自动保存摘要
```

### 3. 检测到时间词时搜索

用户输入包含以下词汇时，主动搜索记忆：

```
昨天、前天、之前、上次、刚才、 earlier、last time
```

**示例：**
- 用户："我们昨天聊到哪了？"
- 动作：自动搜索 memory/ 目录，找到昨天的笔记

### 4. 模型切换检测

如果检测到当前是新模型（无本次会话历史）：

```
1. 提示用户："刚切换模型，正在加载历史记忆..."
2. 自动读取最近的记忆文件
3. 简要总结给用户
```

## 记忆文件结构

```
workspace/
├── MEMORY.md                 # 长期记忆（跨会话、跨模型）
├── memory/
│   ├── 2026-03-24.md        # 每日笔记
│   ├── 2026-03-23.md
│   └── memory-index.json    # 索引文件，便于快速搜索
└── ...
```

## 记忆内容分类

| 类型 | 存储位置 | 内容示例 |
|------|----------|----------|
| 每日对话 | memory/YYYY-MM-DD.md | 当天的讨论、决策 |
| 长期记忆 | MEMORY.md | 用户偏好、重要项目 |
| 快速索引 | memory-index.json | 关键词、时间、主题索引 |

## 脚本工具

### scripts/auto_save.js

自动保存对话摘要：

```bash
# 在对话结束时调用
node scripts/auto_save.js --session-id xxx --summary "讨论了 xxx"
```

### scripts/load_memory.js

加载历史记忆：

```bash
# 会话启动时调用
node scripts/load_memory.js --days 3
# 返回最近 3 天的记忆摘要
```

### scripts/search_memory.js

搜索记忆：

```bash
node scripts/search_memory.js "关键词"
# 返回匹配的记忆片段
```

## 使用示例

**场景 1：用户切换模型后继续对话**
```
用户（新模型）："继续昨天的话题"
→ 检测到 "昨天"
→ 搜索 memory/2026-03-23.md
→ 找到昨天内容
→ 回复："昨天我们讨论了 xxx，要继续吗？"
```

**场景 2：自动保存**
```
对话中用户做出决策
→ 自动追加到 memory/2026-03-24.md
→ 同时更新 memory-index.json
```

## 注意事项

1. **不要过度保存** - 只记关键信息，避免文件膨胀
2. **定期整理** - 每周回顾 MEMORY.md，删除过时内容
3. **隐私保护** - 敏感信息不写入记忆文件
